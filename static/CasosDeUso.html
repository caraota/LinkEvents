  2. En tanto que cualquier persona registrada puede iniciar sesión como usuario para utilizar la aplicación.

    • URL: '/LinkEvents/AIniciarSesion'
    • Método: POST
    • Formato de entrada y salida: JSON
    • Parámetros POST:
      • username - Requerido.
      • password - Requerido.
    • Roles: Sin restricción de rol
    • Requisitos: No disponible

    • Cursos posibles

      1. Precondición: Usuario válido.

        Postcondicion: Sesión iniciada.

        resultado:

        • label:'/VPrincipal',
        • msg:"Sesión iniciada".

        Se continúa con el rol de usuario.

      2. Precondición: Error en datos ingresados.

        Postcondicion: No pasa nada.

        resultado:

        • label:'/VInicioSesion',
        • msg:"Error al iniciar sesión".

        Postcondicion: Usuario registrado exitosamente.

        resultado:

        • label:'/VIniciarSesion',
        • msg:"Usuario registrado".

      2. Precondición: Datos del usuario inválidos.

        Postcondicion: No ocurre nada.

        resultado:

        • label:'/VRegistrarUsuario',
        • msg:"Error al registrar usuario".

    • Roles: admin

    • Cursos posibles

        resultado:

        • msg:"Asistencia confirmada".

      2. Precondición: El admin indica que el participante asistió al evento.

        Postcondicion: Ocurre un error.

        resultado:

        • label:'/VListarUsuarios',
        • msg:"Error".
